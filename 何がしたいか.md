# AtCoderのレーティングを表示するウィジェット

## 概要
- htmlにオブジェクトとして埋め込める
- ウィジェットをクリックしたらAtCoderの当該ユーザーのページに飛べる
- Algorithm, Heuristic の両方に対応

## 大まかなレイアウト
- ユーザ名

## レーティングの取得方法
- 以後、例に挙げるユーザのユーザ名を`user-name`とする

https://atcoder.jp/users/user-name/history/json や https://atcoder.jp/users/user-name/history/json?contestType=heuristic で取得できるjsonの最後尾の要素のNewRatingから取得することができる。

jsonの例(この人のレーティングは22):
```
[{"IsRated":true,"Place":8119,"OldRating":0,"NewRating":9,"Performance":78,"InnerPerformance":78,"ContestScreenName":"abc379.contest.atcoder.jp","ContestName":"トヨタ自動車プログラミングコンテスト2024#11(AtCoder Beginner Contest 379)","ContestNameEn":"Toyota Programming Contest 2024#11（AtCoder Beginner Contest 379）","EndTime":"2024-11-09T22:40:00+09:00"},{"IsRated":true,"Place":9500,"OldRating":9,"NewRating":22,"Performance":-98,"InnerPerformance":-98,"ContestScreenName":"abc380.contest.atcoder.jp","ContestName":"AtCoder Beginner Contest 380","ContestNameEn":"","EndTime":"2024-11-16T22:40:00+09:00"}]
```

## スクリプト
- PythonのスクリプトをGitHub Actionsで毎日15時37分(UTC)に実行してhtmlを書き出す
- ユーザ名は`users-list`から読み取る
  - ユーザ名が改行区切りで複数書かれているファイル
- ただし、`#` から始まる行はコメントであるため読み飛ばす
- 各ユーザに対し、以下を実行
  - 前述のjson2つを取得
  - AtCoderのサーバーにリクエストを送るたびに1秒待つ
  - htmlファイル`./widgets/user-name.html`にウィジェットを書き出す。
  - 得られるJSONが空なら、黒色で`No Rating` とする

## ウィジェットについて
以下の要素を含む
- ユーザ名
- Algorithmレーティング
- Heuristicレーティング
- ウィジェットの更新日時
- ウィジェットの全領域について、クリックしたらAtCoderの当該ユーザーのページに飛べる
  - ただし、Heuristicの部分をクリックしたら`https://atcoder.jp/users/user-name?contestType=heuristic` に飛ぶようにする

## レーティングの色について
AtCoderのと基本的に同じだが、任意色も赤色として扱う
- 2800-: 赤色
- 2400-2799: 橙色
- 2000-2399: 黄色
- 1600-1999: 青色
- 1200-1599: 水色
- 800-1199: 緑色
- 400-799: 茶色
- 1-399: 灰色
- 0: 黒色
